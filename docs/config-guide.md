## 游戏窗口配置

### window_title
**类型**：字符串  
**默认值**：`"杖剑传说"`  
**说明**：游戏窗口的标题，用于自动定位游戏窗口  
**调优建议**：
- 确保与游戏窗口标题完全一致（区分大小写）
- 可通过任务管理器或窗口属性查看精确标题
- 如果标题包含动态变化，可使用部分匹配（如使用 `pygetwindow` 的模糊匹配功能）

### window_region（可选）
**类型**：数组 `[left, top, width, height]`  
**默认值**：注释（自动检测）  
**说明**：手动指定游戏窗口区域，当自动检测失败时使用  
**调优建议**：
- 使用截图工具获取窗口精确坐标
- 仅在自动检测不稳定时启用
- 格式示例：`[658, 34, 564, 1004]`

## 模板路径配置

### sample_dir
**类型**：字符串  
**默认值**：`"assets/samples"`  
**说明**：地图定位器样本图片目录  
**调优建议**：
- 准备多张不同角度、不同位置的定位器截图
- 建议至少 5-10 张样本
- 图片格式：PNG 或 JPG

### battle_button_path
**类型**：字符串  
**默认值**：`"assets/templates/battle_button.png"`  
**说明**：战斗按钮模板图片路径  
**调优建议**：
- 截取完整的战斗按钮（包含边缘和背景）
- 确保按钮处于未激活状态
- 建议使用 PNG 格式保留透明度

## 匹配阈值配置

### confidence_map
**类型**：浮点数（0.0-1.0）  
**默认值**：`0.6`  
**说明**：地图定位器的匹配置信度阈值  
**调优建议**：
- **过低（<0.5）**：匹配率高，但可能误匹配相似物体
- **适中（0.6-0.8）**：推荐范围，平衡准确率和召回率
- **过高（>0.8）**：准确率高，但可能漏检

**场景调优**：
- 复杂背景：提高阈值至 0.7-0.8
- 简单背景：降低阈值至 0.5-0.6
- 高分辨率：可适当降低阈值

### confidence_battle
**类型**：浮点数（0.0-1.0）  
**默认值**：`0.8`  
**说明**：战斗按钮的匹配置信度阈值  
**调优建议**：
- 战斗按钮误操作后果严重，建议保持较高阈值（≥0.8）
- 如果按钮变化较大（如不同状态），可降低至 0.7

### scale_range
**类型**：数组 `[min_scale, max_scale]`  
**默认值**：`[0.8, 1.2]`  
**说明**：多尺度匹配的缩放范围  
**调优建议**：
- 范围过小：无法适配界面缩放
- 范围过大：增加计算量，降低性能
- 根据实际分辨率变化调整，常用范围：`[0.9, 1.1]` 或 `[0.8, 1.2]`

### scale_steps
**类型**：整数  
**默认值**：`5`  
**说明**：多尺度匹配的步数（缩放次数）  
**调优建议**：
- 步数越多，匹配精度越高，但性能越低
- 推荐：3-7 步
- 性能优先：3-4 步
- 精度优先：6-8 步

## 操作参数配置

### click_offset
**类型**：整数（像素）  
**默认值**：`5`  
**说明**：点击时的随机偏移范围（±offset 像素）  
**调优建议**：
- **防检测**：增大偏移量（8-15 像素）
- **精确操作**：减小偏移量（2-5 像素）
- 默认值已足够应对大部分场景

### click_delay
**类型**：数组 `[min_delay, max_delay]`（秒）  
**默认值**：`[0.1, 0.5]`  
**说明**：点击前的随机延迟时间  
**调优建议**：
- **防检测**：增大范围 `[0.2, 0.8]`
- **快速操作**：减小范围 `[0.05, 0.2]`
- 单位：秒

### walk_delay
**类型**：数组 `[min_delay, max_delay]`（秒）  
**默认值**：`[1.0, 3.0]`  
**说明**：走路移动后的等待时间  
**调优建议**：
- 根据游戏移动速度调整
- 快速移动：`[0.5, 1.5]`
- 慢速移动：`[2.0, 4.0]`

### battle_delay
**类型**：数组 `[min_delay, max_delay]`（秒）  
**默认值**：`[4.0, 6.0]`  
**说明**：战斗等待时间  
**调优建议**：
- 根据战斗持续时间调整
- 短战斗：`[2.0, 4.0]`
- 长战斗：`[5.0, 8.0]`

### wait_delay
**类型**：数组 `[min_delay, max_delay]`（秒）  
**默认值**：`[2.0, 4.0]`  
**说明**：场景加载等待时间  
**调优建议**：
- 快速加载：`[1.0, 2.0]`
- 慢速加载：`[3.0, 5.0]`

### loop_delay
**类型**：数组 `[min_delay, max_delay]`（秒）  
**默认值**：`[2.0, 4.0]`  
**说明**：主循环的延迟时间  
**调优建议**：
- 降低延迟可提高响应速度，但增加 CPU 占用
- 推荐：`[1.5, 3.0]`

## 寻路参数配置

### pathfind_offset
**类型**：整数（像素）  
**默认值**：`60`  
**说明**：点击地图定位器下方的偏移量  
**调优建议**：
- 根据游戏界面调整
- 通常在 40-80 像素之间
- 过小：可能点击到定位器本身
- 过大：可能点击范围超出地图

## 异常处理配置

### max_failed_attempts
**类型**：整数  
**默认值**：`3`  
**说明**：连续失败次数阈值，达到后切换到盲走探索  
**调优建议**：
- 探索密集区域：增大至 4-6
- 探索稀疏区域：减小至 2-3
- 0 表示禁用盲走探索

### log_stats_interval
**类型**：整数（秒）  
**默认值**：`300`  
**说明**：统计信息输出间隔  
**调优建议**：
- 实时监控：设置为 60 秒
- 定期报告：设置为 300 秒（5分钟）
- 禁用：设置为 0

## 性能优化配置

### enable_template_cache
**类型**：布尔值  
**默认值**：`true`  
**说明**：是否启用模板缓存  
**调优建议**：
- 始终保持开启（除非调试模板加载）
- 可显著减少 IO 操作

### match_threads
**类型**：整数  
**默认值**：`0`  
**说明**：多尺度匹配的并行线程数  
**调优建议**：
- 0：单线程（默认）
- >0：多线程（需 CPU 支持）
- 通常单线程已足够

## 调试配置

### debug_mode
**类型**：布尔值  
**默认值**：`false`  
**说明**：调试模式，输出详细日志  
**调优建议**：
- 问题排查时启用
- 正常运行时关闭（减少日志量）

### save_screenshots
**类型**：布尔值  
**默认值**：`false`  
**说明**：保存截图用于调试  
**调优建议**：
- 仅在问题排查时启用
- 会占用大量磁盘空间
- 截图保存在 `screenshot_dir` 指定的目录

### screenshot_dir
**类型**：字符串  
**默认值**：`"debug_screenshots"`  
**说明**：截图保存目录  
**调优建议**：
- 使用相对路径（相对于项目根目录）
- 定期清理旧截图

## 常见问题排查

### 问题 1：匹配失败率过高
**可能原因**：
- 匹配阈值过高
- 模板图片不准确
- 游戏分辨率变化

**解决方案**：
1. 降低 `confidence_map` 和 `confidence_battle`
2. 更新 `assets/` 目录下的模板图片
3. 检查 `scale_range` 是否覆盖实际缩放范围

### 问题 2：误匹配频繁
**可能原因**：
- 匹配阈值过低
- 模板图片与实际差异过大

**解决方案**：
1. 提高 `confidence_map` 和 `confidence_battle`
2. 更换更精确的模板图片
3. 减小 `scale_range` 缩放范围

### 问题 3：操作无反应
**可能原因**：
- 窗口定位错误
- 点击偏移过大
- 游戏窗口未激活

**解决方案**：
1. 检查 `window_title` 和 `window_region`
2. 减小 `click_offset`
3. 确保游戏窗口处于前台

### 问题 4：被游戏检测到
**可能原因**：
- 操作过于规律
- 延迟时间过短
- 点击过于精确

**解决方案**：
1. 增大 `click_offset` 至 10-15 像素
2. 延长所有延迟时间
3. 增大 `loop_delay` 降低操作频率
4. 避免长时间连续运行

## 性能优化建议

### 1. 减少模板数量
- 只保留关键的定位器样本
- 移除重复或相似的模板

### 2. 优化缩放匹配
- 减小 `scale_steps`（如从 5 降至 3）
- 缩小 `scale_range`（如 `[0.9, 1.1]`）

### 3. 调整延迟参数
- 在保证功能的前提下适当降低延迟
- 避免不必要的长时间等待

### 4. 关闭调试功能
- 确保 `debug_mode` 和 `save_screenshots` 为 `false`

---

## 防检测最佳实践

1. **随机化一切**：所有延迟、偏移、移动都使用随机值
2. **模拟真人节奏**：操作速度与真人接近，避免机械重复
3. **适度休息**：运行一段时间后暂停，避免长时间连续操作
4. **降低操作频率**：增大 `loop_delay`，减少每分钟的操作次数
5. **多样化行为**：结合盲走探索，避免固定路径

---

**注意**：修改配置后，需要重启 Agent 才能生效。
